# cartoon-plants

Контент-завод для мультяшных растений: 4 культуры — **томат, клубника, огурец, картофель**.
Короткие ролики про уход за огородом для ЦА 35–60 лет (женщины): динамичные идеи → проверка фактов → 5 сцен-картинок → 5 видео → склейка.

Архитектура сознательно повторяет `veo-scripts` (только API-подключения), но:
- **агенты** на OpenRouter (идея → выбор → сценарий → **фактчек точности** → промпты);
- **картинки** через PoYo `gpt-image-2-edit` от канонических референсов-изображений;
- **видео** через VeoNonStop (`/video/image-to-video`);
- **референсы персонажа = картинки** (текстовый `character_bible` убран — источником правды служит изображение);
- **все ключи через env / `.env`**, в коде их нет.

## Что делает

```
задание (crop) → референс персонажа (image)
   → [OpenRouter] brainstorm идей → выбор лучшей
   → [OpenRouter] сценарий 5 сцен (с claim-ами)
   → [OpenRouter] ФАКТЧЕК агрономической точности → исправленный сценарий
   → [OpenRouter] 5 image-промптов + 5 video-промптов
   → [PoYo gpt-image-2-edit] 5 сцен-картинок (референс персонажа на входе)
   → [VeoNonStop] 5 видео (image-to-video)
   → [ffmpeg] склейка в один ролик 1080×1920 30fps
   → (опц.) Google Drive / Telegram
```

## Запуск

```bash
npm install
cp .env.example .env      # заполнить ключи

# Проверка связи картинок+видео на одной сцене (без агентов, перед полным прогоном):
node selftest.js --crop tomato

# Только агенты (идеи→фактчек→промпты), дёшево:
node run.js --crop tomato --only agents

# Весь пайплайн: агенты → картинки → видео → склейка
node run.js --crop tomato
```

Культуры: `tomato`, `strawberry`, `cucumber`, `potato`.

## Структура

```
run.js                 — оркестратор пайплайна
lib/
  util.js              — sleep, extractJson, requestBuffer, загрузка
  openrouter.js        — клиент OpenRouter (агенты/текст)
  poyo-image-gpt2.js   — клиент PoYo (картинки) + upload референса
  veononstop.js        — клиент VeoNonStop (видео)
  ffmpeg.js            — склейка/аудио
  google.js            — Google Drive (опц.)
agents/
  pipeline.js          — цепочка агентов: идея→выбор→сценарий→фактчек→промпты
  prompts.js           — системные промпты агентов
crops/                 — конфиг культур (голос, банки тем, среда — БЕЗ описания персонажа)
  tomato.json strawberry.json cucumber.json potato.json
references/            — локальный кэш (скачивается из Google Drive, в репо нет)
output/                — сгенерённое (в .gitignore)
```

## Безопасность

- Ключи **только** в `.env` (в `.gitignore`). В коде их нет.
- **Референсы персонажа живут в Google Drive** (папка `cartoon-plants-references`); в репо их нет. `run.js` сам скачивает нужный референс по `reference_url` из конфига культуры в локальный кэш `references/`.

## Отличия от veo-scripts

- Добавлен **фактчек-агент** (точность советов по агрономии — ЦА не прощает вранья).
- Динамическая генерация идей (brainstorm нескольких + выбор лучшего) вместо зашитого жанра.
- Референс персонажа — изображение, а не текстовое описание.
- Auth через env вместо хардкода ключей в `lib/*.js`.
